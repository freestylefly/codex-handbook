# Desktop App

Desktop App 更像一个本地 Codex 工作台。它适合在一个项目里管理多条任务线：你可以让不同 agent 并行探索、实现、验证，也可以用 Skills 和 Automations 固化团队流程。

## 适用场景

- 同时推进多个本地任务。
- 让 agent 在本地工作树里完成更长的实现。
- 使用 Skills 复用复杂工作流。
- 使用 Automations 做提醒、监控或后续跟进。
- 借助插件连接浏览器、文档、表格、演示文稿等工作场景。

## 学习重点

- 工作区与 Git 状态：每次动手前确认当前分支和未提交改动。
- agent 分工：只把边界明确、可独立验证的任务交给并行 agent。
- Skills：把重复流程写成项目或个人能力。
- Automations：把周期性任务和延迟任务交给系统。
- 插件：面向浏览器、文档、表格、演示等非纯代码任务扩展能力。

## 实践建议

- 先把任务拆成“探索、实现、验证”。
- 并行任务要有明确文件边界。
- 长任务要要求阶段性总结。
- 涉及删除、迁移、发布、生产资源时保留人工确认。
- 成功案例及时沉淀到 `docs/recipes/` 或团队知识库。
